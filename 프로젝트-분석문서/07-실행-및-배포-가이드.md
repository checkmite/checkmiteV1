# 07. 실행 및 배포 가이드

## 1. 사전 준비

| 도구 | 용도 |
|------|------|
| Node.js + npm | 프론트엔드 / 백엔드 |
| Python 3 (+venv, pip) | AI 모델 서버 |
| PostgreSQL | 데이터베이스 |
| (Linux) libgl1 | OpenCV 의존성 |

---

## 2. 의존성 설치

```bash
# 프론트 + 백엔드 (Node)
npm install

# 모델 서버 (Python) — venv 생성 + 패키지 설치
npm run api:install
# 또는 수동:
python3 -m venv .venv
. .venv/bin/activate        # Windows: .venv\Scripts\activate
python -m pip install -r api/requirements.txt
```

---

## 3. 환경변수 설정

```bash
cp .env.example .env        # Windows PowerShell: Copy-Item .env.example .env
```

### 기본 `.env`
```env
NODE_ENV=development
PORT=3000
DATABASE_URL=postgresql://checkmite:password@127.0.0.1:5432/checkmite
UPLOAD_DIR=uploads
UPLOAD_MAX_IMAGE_MB=20
UPLOAD_MAX_VIDEO_MB=500
MODEL_RUNTIME_URL=http://127.0.0.1:8000
MODEL_RUNTIME_REQUIRED=false
FRONTEND_DIST_DIR=dist
```

> `DATABASE_URL`은 실제 PostgreSQL 계정/비밀번호/DB명에 맞게 수정.
> `MODEL_RUNTIME_REQUIRED=false`면 모델 서버 없이 fallback 결과로 개발 가능.

## 4. 로컬 개발 실행

### (필수) DB 테이블 생성
```bash
npm run backend:migrate
```

### 프로세스별 실행 (터미널 3~4개)
```bash
# 1) 모델 서버 (8000)
npm run api
#   = uvicorn api.model_server:app --host 0.0.0.0 --port 8000

# 2) 백엔드 (3000)
npm run backend:dev
#   = node backend/src/server.js

# 3) 프론트엔드 (개발)
npm run dev
#   = vite --host 0.0.0.0 --port 80   ← package.json 기준 80포트
```

### 접속 주소
```
Frontend : http://localhost:5173 (README 기준) / 80 (package.json 기준)
Backend  : http://localhost:3000/api
Model    : http://localhost:8000
```

> ⚠️ `package.json`의 dev 스크립트는 `--port 80`, README는 `5173`로 안내. 환경에 맞게 확인 필요.

### 상태 확인
```bash
curl http://localhost:3000/api/health      # 백엔드 + DB
curl http://localhost:8000/health          # 모델 서버 + 가중치
# Windows: Invoke-WebRequest http://localhost:3000/api/health
```

### 모델 단독 테스트
```bash
curl -F "file=@sample.jpg" "http://localhost:8000/predict/image?conf=0.25&imgsz=640"
```

## 5. 빌드

### 표준 빌드
```bash
npm run build      # tsc -b && vite build → dist/
```

### minify 실패 시 분리 검증 (README 안내)
```bash
# 타입 검사만
npx tsc -b --pretty false

# minify 없이 번들
node ./node_modules/vite/bin/vite.js build --minify=false --emptyOutDir=false
```

---

## 6. 단일 서버로 프론트까지 서빙

`FRONTEND_DIST_DIR=dist` 설정 후:
```bash
node ./node_modules/vite/bin/vite.js build --minify=false --emptyOutDir=false
npm run backend:dev
```
→ Express가 `/api`와 정적 프론트(`dist/`)를 함께 서빙. SPA 폴백 포함.

## 7. 운영 배포 구조

```
nginx (80/443)
  ├─ React/Vite 정적 파일 서빙
  └─ /api → Express 프록시

Express Backend (3000)   — REST API, DB, 모델 호출
Model Runtime (8000)     — detection / density / vitality
PostgreSQL (5432)        — 데이터 저장
```

### 프로세스 관리 (택1)
```
systemd:
  checkmite-backend.service
  checkmite-model-runtime.service
  postgresql.service
  nginx.service

또는 Docker Compose:
  services: nginx / backend / model-runtime / postgres
```

### 운영 권장 설정
```env
NODE_ENV=production
MODEL_RUNTIME_REQUIRED=true     # 모델 실패 시 fallback 금지
UPLOAD_DIR=/srv/checkmite/uploads
FRONTEND_DIST_DIR=/srv/checkmite/frontend/dist
```

## 9. 주의점 / 알려진 한계

| 항목 | 내용 |
|------|------|
| 진행률 인메모리 | `analysis-progress.service`는 `Map` 기반 → 서버 재시작 시 진행중 작업 소실, 다중 인스턴스 미지원 |
| dev 포트 불일치 | package.json(80) vs README(5173) |
| 빌드 minify | 환경에 따라 Vite minify 단계가 실패할 수 있음 (분리 빌드로 회피) |
| 문서 vs 구현 | 설계 문서는 면적당 밀도(cm²), 실제 코드는 L당 밀도(영상 카운트) 기준 |
| GPU | YOLOv8 추론 성능은 GPU 유무에 크게 좌우 (CPU도 동작은 함) |
| 모델 가중치 | 최초 추론 시 `best.pt.part-*` 자동 병합 → 첫 요청 지연 가능 |
