# 06. 탭별 기능 상세

5개 탭 각각의 **화면 / 사용 기술 / 데이터 흐름 / 구동 원리**.

공통 전제: 모든 분석 탭은 `selectedBoxId`(사육박스)를 필수 컨텍스트로 가지며, `PhaseId` 상태머신(`idle → file → proc → result`)을 따른다.

---

## 1. 객체 탐지 (DetectionPage)

| 항목 | 내용 |
|------|------|
| 목적 | 이미지 1장에서 천적·먹이응애 개체 탐지 |
| 입력 | 이미지 (JPG/PNG/WEBP, 최대 20MB) |
| API | `POST /api/analysis/detection` (multipart, **동기**) |
| 모델 | YOLOv8 타일 추론 |

### 데이터 흐름
```
파일 선택 → "탐지 실행"
  → api.analyzeDetection(boxId, file)  (Promise를 useRef에 저장)
  → 동시에 Processing 가짜 애니메이션(2.6초) 재생
  → 애니메이션 종료(onAnimDone) 시점에 API 결과 반영
  → 박스 좌표(0~1) → ×100 → % 단위 오버레이 렌더
```

### 구동 원리 포인트
- **연출/실제 분리**: 빠른 추론이라 실제 진행률 대신 애니메이션을 보여주고, 백그라운드에서 진짜 API를 호출
- **반응형 박스**: 정규화 좌표를 `%`로 그려 이미지 리사이즈에 자동 대응
- **결과 지표**: 천적/먹이 마릿수, 비율 바, 개체별 confidence 막대, 평균 신뢰도

### 화면 단계
```
idle  : UploadZone + "이렇게 동작합니다" 안내
file  : FileChip + "탐지 실행" 버튼
proc  : Processing (DET_STEPS 5단계 텍스트)
result: 바운딩박스 이미지 + 통계 카드
```

## 2. 밀도 측정 (DensityPage)

| 항목 | 내용 |
|------|------|
| 목적 | 영상 기반 밀도 + 활력도 **통합 분석** |
| 입력 | 1mL 배지 영상 N개 (각 10초↑, 최대 500MB) |
| API | `POST /api/analysis/density` (**비동기 202**) + 진행률 폴링 |
| 모델 | 프레임 카운트(밀도) + 추적/속도(활력도) |

### 데이터 흐름 (가장 복잡)
```
영상 다중 선택 (중복 제거: name-size-lastModified 키)
  → "통합 분석 시작"
  → [클라이언트 검증] getVideoDuration()으로 각 영상 10초 확인
  → api.startDensityAnalysis()  (XHR, 업로드 % 콜백)
  → jobId 수신
  → setInterval 1초마다 getDensityProgress(jobId)
       · percent / message / 샘플별 status 갱신
       · status==='completed' → 결과 표시
       · status==='failed'    → 에러
```

### 구동 원리 포인트
- **클라이언트 사전검증**: 업로드 전 `<video>` 메타데이터로 길이 확인 → 서버 왕복 절약
- **실제 진행률**: `Processing`에 폴링 percent 전달 → 가짜 애니메이션 대신 진짜 진행
- **샘플별 실시간 표시**: 각 영상의 `density → vitality → completed` 단계와 중간 수치(마리/mL, 점수, tracks) 노출
- **2단계 추론**: 같은 영상으로 밀도(카운트)와 활력도(추적)를 순차 실행

### 결과 화면 구성
```
증식 활력 대표지표  : 속도비 + 위험/주의/정상 경고 배너
3-스탯            : 현재 count(마리/mL) · 활력도(점) · 활동비율(%)
밀도 정보         : L당 밀도, 평균 count, 분석 영상 수, 경고
활력도 세부       : 평균속도(mm/s), 속도비(x), 확정/이동 track
트래킹 영상       : 1번 영상 추적 시각화 <video>
```

### 경고 임계값
| 속도비 | 등급 | 색상 |
|--------|------|------|
| ≤ 2x | 위험 | 빨강 |
| ≤ 3x | 주의 | 노랑 |
| > 3x | 정상 관찰 | 초록 |

## 3. 증식률 분석 (GrowthPage)

| 항목 | 내용 |
|------|------|
| 목적 | 측정 이력 기반 증식 추세 분석 |
| 입력 | **없음** (선택 박스 누적 이력) |
| API | `GET .../growth` + `GET .../measurements` (병렬) |
| 모델 | **추론 없음** (DB 집계) |

### 데이터 흐름
```
탭 진입 / 박스 변경
  → Promise.all([ api.getGrowth(boxId), api.listMeasurements(boxId) ])
  → growth(집계 결과) + measurements(원시 이력) 상태 저장
  → 카드/차트 렌더
```

### 구동 원리 포인트
- **2가지 증식률**: 첫 측정 대비 / 직전 측정 대비
- **가중 통합**: `count변화율 × 0.75 + 활력도변화율 × 0.25`
- **회차 병합**(`compactAnalysisHistory`): 5분 이내 측정들을 한 "통합 분석" 회차로 묶어 표시
- **자체 SVG 차트**(`LineChart`): "기준 대비 증식률(%)" 추이 그래프
- **확장 카드**(expand): 카드 클릭 시 세부 지표 펼침

### 화면 구성
```
대표 증식률 hero  : 라벨(증식 활발/감소 추세/유지) + 기간
4개 서브카드      : 현재 마리/1L · 증식률 그래프 · 직전 대비 · 첫 측정 대비
분석 이력 테이블  : 날짜 · 분석유형 · 마리/1L · 활력도
```

### 라벨 판정 (`growthStatus`)
| 직전 통합 증식률 | 라벨 |
|------------------|------|
| > 20% | 최근 증식 활발 |
| < -10% | 최근 감소 추세 |
| 그 외 | 최근 유지 관찰 |
| 비교 데이터 < 2회 | 비교 데이터 부족 |

## 4. 사육박스 관리 (CultureBoxesPage)

| 항목 | 내용 |
|------|------|
| 목적 | 사육박스 등록·수정·삭제(CRUD) |
| API | `GET/POST/PATCH/DELETE /api/culture-boxes` |

### 화면 구성 (3분할)
```
목록 카드   : 전체 박스 (시작일 내림차순), 선택 표시
등록 카드   : 이름 / 시작일 / 메모 → 새 박스 생성
수정 카드   : 선택 박스 편집 + "휴지통으로 이동" 버튼
삭제 모달   : 확인 다이얼로그
```

### 구동 원리 포인트
- **제어 컴포넌트 폼**: createForm / editForm 분리 관리
- **낙관적 업데이트**: API 성공 시 App.tsx 로컬 state 즉시 반영
- **soft delete**: 삭제 = "휴지통으로 이동" (물리 삭제 아님)
- **최소 1개 보장**: 마지막 박스는 삭제 버튼 비활성화
- **초기 개체수 없음**: 이름/시작일/메모만 (설계 원칙)

## 5. 휴지통 (TrashPage)

| 항목 | 내용 |
|------|------|
| 목적 | 삭제된 박스 보관·복구 |
| API | `GET /api/trash/culture-boxes`, `POST .../:id/restore` |

### 구동 원리 포인트
- `deleted_at`이 채워진 박스 목록 표시 (삭제 시각, 측정 이력 개수)
- **복구**: `deleted_at = NULL`로 되돌려 박스 + 측정 이력 전체 부활
- 복구 시 App.tsx에서 활성 목록에 다시 추가하고 휴지통에서 제거 (낙관적)
- 빈 상태 안내 UI 제공

## 6. 탭 간 공통 패턴 요약

| 패턴 | 적용 탭 | 설명 |
|------|---------|------|
| `PhaseId` 상태머신 | 탐지, 밀도 | idle→file→proc→result |
| 가짜 진행 애니메이션 | 탐지 | 빠른 동기 추론 |
| 실제 폴링 진행률 | 밀도 | 느린 비동기 추론 |
| 낙관적 업데이트 | 박스관리, 휴지통 | API 성공 후 로컬 즉시 갱신 |
| BoxSelector 즉석 생성 | 탐지, 밀도, 증식률 | 분석 중 박스 바로 추가 |
| localStorage 영속 | 전체 | 탭/테마/선택박스 복원 |
