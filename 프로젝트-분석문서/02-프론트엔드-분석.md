# 02. 프론트엔드 분석

## 1. 기술 스택

| 항목 | 기술 | 비고 |
|------|------|------|
| 프레임워크 | React 18.3 | 함수형 컴포넌트 + Hooks |
| 빌드 도구 | Vite 6 | 개발서버 + 번들러 |
| 언어 | TypeScript 5.6 | strict 타입 |
| 라우팅 | **없음** | `tab` state로 조건부 렌더링 |
| 상태관리 | **없음** (Redux/Zustand 미사용) | `useState` + props drilling |
| 스타일 | 순수 CSS + CSS 변수(디자인 토큰) | 컴포넌트별 `.css` 파일 |
| HTTP | `fetch` + `XMLHttpRequest` | 업로드 진행률용 XHR |

> 외부 UI 라이브러리·차트 라이브러리 없이 **자체 제작**. 의존성이 매우 가볍다.

## 2. 진입점과 루트 (`App.tsx`)

### 역할
- 전역 상태(사육박스 목록, 휴지통, 선택 박스, 테마, 탭)를 보유하고 모든 탭에 props로 내려줌
- 최초 마운트 시 박스/휴지통 병렬 로드
- 사육박스 CRUD 콜백을 정의해 탭에 전달 (낙관적 업데이트)

### 핵심 상태
```ts
const [boxes, setBoxes]           // 활성 사육박스 목록
const [trashDb, setTrashDb]       // 휴지통 박스 목록
const [tab, setTab]               // 현재 탭 (localStorage 'cm-tab')
const [theme, setTheme]           // light/dark (localStorage 'cm-theme')
const [selectedBoxId, ...]        // 선택 박스 (localStorage 'cm-box-id')
```

### localStorage 활용
| 키 | 용도 |
|----|------|
| `cm-tab` | 마지막 탭 복원 (`vitality`는 `density`로 자동 매핑) |
| `cm-theme` | 라이트/다크 테마 |
| `cm-box-id` | 마지막 선택 박스 |

### 탭 라우팅 방식
```tsx
{tab === 'detection' && <DetectionPage ... />}
{tab === 'density'   && <DensityPage ... />}
{tab === 'growth'    && <GrowthPage ... />}
{tab === 'boxes'     && <CultureBoxesPage ... />}
{tab === 'trash'     && <TrashPage ... />}
```
> URL이 바뀌지 않는 SPA. 새로고침 시 `localStorage`로 탭 복원.

### 사육박스 CRUD 패턴 (낙관적 업데이트)
- 추가/수정/삭제 모두 `api.*` 호출 성공 후 **로컬 state를 즉시 갱신** → 재요청 없이 UI 반영
- 삭제 시: 활성 목록에서 제거 + 휴지통 목록에 추가 (서버 round-trip 최소화)
- 마지막 박스 1개는 삭제 차단 (`boxes.length <= 1`)

## 3. API 클라이언트 (`src/api/client.ts`)

모든 백엔드 호출을 한 파일에 모은 얇은 래퍼 + 응답 타입 정의.

### 공통 요청 함수
```ts
async function req<T>(path, options): Promise<T> {
  const res = await fetch(`/api${path}`, options);
  if (!res.ok) throw new Error(body.message ?? `HTTP ${res.status}`);
  return res.json();
}
```

### 제공 메서드
| 메서드 | 엔드포인트 | 비고 |
|--------|-----------|------|
| `listBoxes` | GET `/culture-boxes` | |
| `createBox / updateBox / deleteBox` | POST/PATCH/DELETE `/culture-boxes/:id` | |
| `listTrash / restoreBox` | `/trash/culture-boxes...` | |
| `listMeasurements` | GET `/culture-boxes/:id/measurements` | |
| `getGrowth` | GET `/culture-boxes/:id/growth` | from/to 쿼리 |
| `analyzeDetection` | POST `/analysis/detection` | multipart, 동기 |
| `startDensityAnalysis` | POST `/analysis/density` | **XHR**, 업로드 진행률 콜백 |
| `getDensityProgress` | GET `/analysis/density/:id/progress` | 폴링 |
| `analyzeVitality` | POST `/analysis/vitality` | 단독 활력도(현재 UI 미사용) |

### 왜 density만 XHR을 쓰나?
`fetch`는 업로드 진행률 이벤트를 지원하지 않음 → 대용량 영상 업로드 % 표시를 위해 `XMLHttpRequest.upload.onprogress` 사용.

## 4. 공통 타입 (`src/types/index.ts`)

```ts
type TabId = 'detection' | 'density' | 'growth' | 'boxes' | 'trash';
type PhaseId = 'idle' | 'file' | 'proc' | 'result';   // 분석 화면 단계

interface CultureBox { id; name; startedAt; memo? }
interface Measurement { id; boxId; type; measuredAt; countValue?;
                        densityPerLiter?; vitalityScore?; activeRatio?; resultJson? }
interface DetectionBox { id; cls: 'predator'|'prey'; conf; x; y; w; h }  // 좌표는 % 단위
```

### `PhaseId` — 분석 탭의 화면 상태머신
```
idle (업로드 대기) → file (파일 선택됨) → proc (분석 중) → result (결과)
```
탐지·밀도 탭이 이 4단계를 공유한다.

## 5. 주요 재사용 컴포넌트

| 컴포넌트 | 역할 |
|----------|------|
| `Nav` | 상단 탭 네비게이션 + 테마 토글 |
| `BoxSelector` | 사육박스 선택 드롭다운 + 즉석 생성 |
| `UploadZone` | 드래그&드롭 이미지 업로드 영역 |
| `FileChip` | 선택된 파일 칩 (이름/용량/제거) |
| `Processing` | 분석 진행률 애니메이션 |
| `LineChart` | 자체 SVG 라인 차트 |
| `Gauge` / `Heatmap` / `Badge` | 게이지·히트맵·뱃지 |
| `Icons` | 인라인 SVG 아이콘 세트 |
| `Placeholder` | 이미지 없을 때 자리표시 |

### `Processing` 컴포넌트의 이중 모드 (핵심)
```ts
if (percent !== undefined) {
  // [실제 모드] 부모가 준 percent를 그대로 표시 (밀도 폴링)
} else {
  // [연출 모드] requestAnimationFrame으로 2.6초간 가짜 진행 → onDone 콜백
}
```
- **탐지 탭**: percent 미전달 → 가짜 애니메이션. 끝나면 미리 시작해 둔 API 결과를 반영
- **밀도 탭**: 실제 폴링 percent 전달 → 진짜 진행률

### `LineChart` (자체 SVG)
- 외부 차트 라이브러리 없이 `viewBox 640×height`에 `path`로 라인+영역 그림
- y축 4개 눈금 자동 생성, 그라디언트 채움
- 증식률 탭에서 "기준 대비 증식률(%)" 추이 표시에 사용

## 6. 결과 시각화 기법

### 바운딩 박스 오버레이 (탐지 탭)
- 모델은 좌표를 정규화(0~1)로 반환 → 프론트에서 `×100` 해 `%` 단위로 변환
- 원본 이미지 위에 `position:absolute` div를 `left/top/width/height = %`로 배치
- 이미지 크기가 바뀌어도 박스가 비율대로 따라감 (반응형)

```tsx
<div className="det-box" style={{ left:b.x+'%', top:b.y+'%', width:b.w+'%', height:b.h+'%' }}>
  <span className="db-tag">{classLabel(b.cls)} {(b.conf*100).toFixed(0)}%</span>
</div>
```

### 트래킹 영상 재생 (밀도 탭)
- 백엔드가 생성한 `.webm` 추적 영상 URL을 `<video controls>`로 직접 재생

## 7. 프론트엔드 특징 정리

1. **무(無)프레임워크 상태관리** — props drilling으로 충분히 단순
2. **localStorage 영속화** — 탭/테마/선택박스를 새로고침에도 유지
3. **클라이언트 사전 검증** — 영상 길이(10초)를 업로드 전에 `<video>` 메타데이터로 확인
4. **연출과 실제의 분리** — 빠른 추론은 애니메이션, 느린 추론은 실제 폴링
5. **순수 CSS 디자인 토큰** — `styles/tokens.css`의 CSS 변수로 라이트/다크 일괄 제어
