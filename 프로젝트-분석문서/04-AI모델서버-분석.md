### 가중치 분할 보관 (핵심 트릭)
GitHub 파일당 100MB 제한 회피를 위해 `best.pt`를 `best.pt.part-aa`, `best.pt.part-ab`로 분할.

```python
def restore_model_from_parts():
    if MODEL_PATH.exists(): return
    parts = sorted(part_dir.glob("best.pt.part-*"))
    with tmp_path.open("wb") as out:
        for part in parts:
            out.write(part.read_bytes())  # 순서대로 이어붙임
    tmp_path.replace(MODEL_PATH)          # 원자적 교체
```
서버 첫 추론 요청 시 `best.pt`가 없으면 자동 병합. 모델은 전역 싱글톤으로 1회만 로드.

## 4. 객체 탐지 — SAHI 스타일 타일 추론

`predict_tiled()` — 작은 응애를 큰 현미경 이미지에서 놓치지 않기 위한 핵심 기법.

### 원리
```
1. 큰 이미지를 640px 타일로 분할 (50% 겹침)
     axis_starts(): stride = tile_size × (1 - overlap)
     마지막 타일은 끝에 정렬해 누락 방지

2. 각 타일을 YOLOv8로 개별 추론
     타일 좌표 → 원본 좌표로 환원 (x1 + x_offset)
     이미지 경계로 clamp

3. 겹친 타일의 중복 탐지 제거
     class_aware_nms(): 같은 클래스 + IoU > 0.3 → 낮은 confidence 제거

4. predator/prey 클래스별 개수 집계
```

### 기본 파라미터
| 파라미터 | 기본값 |
|----------|--------|
| confidence | 0.5 |
| imgsz | 640 |
| tile_size | 640 |
| overlap | 0.5 |
| nms_iou | 0.3 |

### 응답 구조
```json
{
  "image": {"width", "height"},
  "settings": {...},
  "counts": {"predator": N, "prey": M},
  "total": N+M,
  "detections": [{"class_name", "confidence", "box": {x1,y1,x2,y2,width,height}}]
}
```

## 5. 밀도 분석 — 영상 프레임 카운트

`analyze_density_video()` — 1mL 배지 영상에서 천적응애 밀도 추정.

### 처리 흐름
```
1. 영상 검증: 최소 10초 이상 (DEFAULT_DENSITY_MIN_VIDEO_SECONDS)

2. 프레임 샘플링: 앞 10초 구간을 1초 간격(sample_stride = fps × interval)

3. 프레임별 처리:
   - YOLOv8 추론 → predator만 카운트
   - frame_quality() 화질 평가

4. 대표 프레임 선정 + 평균 계산:
   - best_frame_count = (count, 화질, -프레임인덱스) 최댓값
   - average_frame_count = 전체 평균
   - estimated_count_per_ml = round(평균 × 배수[기본 10])
   - density_per_liter = mL당 × 1000

5. 등급: estimated_count_per_ml >= 10 → "marketable", 미만 → "low"
```

### 화질 평가 (`frame_quality`)
흐리거나 너무 어둡/밝은 프레임을 거르는 품질 게이트.
```python
sharpness  = Laplacian(gray).var()        # 라플라시안 분산 = 선명도
brightness = gray.mean()                   # 평균 밝기
quality_score = 0.65 × sharpness_score + 0.35 × brightness_score
passes_quality = sharpness ≥ 50 AND 40 ≤ brightness ≤ 220
```
품질 통과 프레임이 하나도 없으면 경고 후 전체에서 best 선택(graceful degradation).

## 6. 활력도 분석 — 추적 + 속도 기반 점수

밀도 영상을 재사용. **2개 모듈**이 협력한다.

```
analyze_vitality_video()
   ├─ TrackingService   (프레임 간 객체 추적 → track_id 부여)
   └─ VitalityService   (track별 속도 → 활력 점수 산출)
```

### 6.1 추적 (`tracking_service.py`) — SORT 유사 알고리즘

프레임마다 탐지된 천적응애를 이전 프레임의 track과 매칭.

```
매칭 기준 (같은 클래스만):
  - 중심거리(center_distance) ≤ 유효이동임계값
  - 또는 IoU ≥ 0.05
  - 후보쌍을 (-IoU, 거리) 정렬 → 그리디 1:1 매칭

유효이동임계값 (_valid_move_threshold):
  bbox 대각선 × 비율 기반, [8px, 60px] 범위로 clamp
  → "텔레포트" 수준의 비현실적 이동은 outlier로 거부

track 갱신:
  total_distance_px 누적, moving_frames(≥2px 이동) 카운트
  trajectory에 위치 이력 저장

미매칭 track: misses++ (4회 초과 시 매칭 후보 제외)
미매칭 detection: 새 track_id 생성
```

| 상수 | 값 | 의미 |
|------|-----|------|
| `MAX_TRACK_MISSES` | 4 | 연속 미검출 허용 |
| `MOTION_THRESHOLD_PX` | 2.0 | 움직임 판정 최소 이동 |
| `MAX_VALID_MOVE_PX` | 60 | 1프레임 최대 허용 이동 |
| `MIN_VALID_MOVE_PX` | 8 | 최소 매칭 반경 |
| `MIN_IOU_FOR_MATCH` | 0.05 | 매칭 최소 IoU |

### 6.2 활력 점수 (`vitality_service.py`)

```
1. 확정 track 필터: frames_seen ≥ 3 AND class == predator

2. track별 속도:
   mean_speed_px_sec = total_distance_px / duration_seconds
   mean_speed_mm_sec = px / px_per_mm(1500)
   speed_ratio = mean_speed_mm_sec / 먹이응애기준속도(0.025mm/s)

3. 속도비 → 점수 (로지스틱 곡선):
   vitality_score = 100 × speed_ratio^k / (speed_ratio^k + midpoint^k)

4. 집계:
   vitality_score(전체) = 평균 속도점수 × 100
   moving_ratio  = 이동 track 비율
   active_ratio  = 활동 개체 비율
```

#### 로지스틱 곡선 자동 피팅 (`_speed_curve_params`) — 핵심
점수를 하드코딩하지 않고 **기준점 2개**로 시그모이드 파라미터를 역산:
```
입력 앵커:  "속도비 3x → 0.60점",  "속도비 6x → 0.92점"
출력:       지수 k, 중점 midpoint  (로그-오즈로 연립방정식 풀이)
```
→ 환경변수로 앵커만 바꾸면 점수 곡선 전체가 부드럽게 재조정됨. 운영 튜닝 친화적.

### 6.3 트래킹 시각화 영상
`renderTrackingVideo=true`면 OpenCV `VideoWriter`(VP80/webm)로:
- 각 객체에 바운딩박스 + `ID n` 라벨
- 최근 30프레임 이동 궤적을 선으로 그림
→ 백엔드 `/uploads/tracking/`에 저장 후 프론트에서 재생.

## 8. 모델 서버 특징 정리

1. **타일 추론(SAHI)** — 고해상도 이미지의 미세 객체 탐지 정확도 확보
2. **가중치 분할 자동복원** — GitHub 용량 제한 우회
3. **품질 게이트** — 흐린/어두운 프레임 자동 제외
4. **물리 기반 활력 측정** — 픽셀→mm 환산 + 기준종 상대속도
5. **로지스틱 자동 피팅** — 앵커 2점으로 점수 곡선 재조정 가능
6. **추론 직렬화 락** — 동시 요청으로부터 모델 보호
